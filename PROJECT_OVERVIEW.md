# ملف شرح موقع بورتفوليو المهندس كمال صرور (Senior Full-Stack & Backend Engineer)

موقع البورتفوليو الشخصي للمهندس **كمال صلاح صرور** هو منصة تعريفية متكاملة مصممة لعرض خبراته المهنية التي تزيد عن 6 سنوات، واستعراض مشاريعه التقنية العميقة، ومهاراته المتقدمة في تطوير الأنظمة الخلفية (Backend) والواجهات الأمامية (Frontend)، مع التركيز بشكل خاص على تقنيات **Python (FastAPI)** و **PHP (Laravel)** ومكاملة أدوات الذكاء الاصطناعي (AI Agents) والبوابات المالية واللوجستية.

الموقع مبني بالكامل كـ **Front-end فقط** بدون سيرفر خاص (No Backend Runtime) لضمان السرعة الفائقة وإمكانية الاستضافة على أي سيرفر مشترك (Shared Hosting / cPanel) أو منصات استضافة المواقع الساكنة مثل GitHub Pages و Cloudflare و Vercel.

---

## 🎯 أهداف الموقع (Objectives)
1. **بناء الهوية الرقمية (Personal Brand):** إبراز الثقل المهني والخبرة الكبيرة (Senior-level) للمهندس كمال صرور أمام أصحاب الأعمال والشركات التقنية العالمية.
2. **استعراض المشاريع (Project Showcase):** تقديم دراسات حالة (Case Studies) مفصلة لأهم خمسة مشاريع قام بتطويرها، توضح المشكلة، الحل الهندسي، والنتائج بالأرقام والنسب المئوية.
3. **عرض المهارات التقنية المنظمة (Tech Stack Showcase):** تقسيم التكنولوجيات بوضوح لتسهيل استيعابها من قِبل مدراء التقنية (CTOs) ومسؤولي التوظيف.
4. **توفير السيرة الذاتية للتحميل (Downloadable Resume):** زر مباشر ومميز لتحميل السيرة الذاتية كملف PDF احترافي وجاهز.
5. **جلب العملاء والفرص (Lead Generation):** نموذج اتصال متكامل يرسل الرسائل مباشرة إلى بريد المهندس كمال الشخصي باستخدام تقنيات الاتصال المباشر من المتصفح (EmailJS).
6. **المدونة التقنية (Tech Blog):** مساحة لنشر المقالات والدروس لتوثيق المعرفة التقنية وإثبات الريادة المعرفية في مجالات الـ Backend وهندسة البرمجيات.

## ⚡ الخصائص والميزات الفنية للموقع (Features)

### 1. واجهة مستخدم تفاعلية ومتميزة (Premium UI/UX)
تصميم مظلم بالكامل مستوحى من لوحات المطورين والأنظمة البرمجية الفخمة (Dark Tech Theme). استخدام مؤثرات حركية خفيفة وسلسة (Micro-animations) لزيادة التفاعل دون إبطاء التصفح.

### 2. التوافق الكامل مع الهواتف والأجهزة (Responsive Design)
الموقع مصمم بشكل كامل ليتوافق مع جميع مقاسات الشاشات:
- شاشات الموبايل (Mobile)
- الأجهزة اللوحية (Tablets)
- شاشات الحواسب المحمولة والمكتبية (Desktops/Laptops)

### 3. دعم كامل لتطبيقات الويب التقدمية (PWA - Progressive Web App)
- إمكانية تثبيت الموقع كأيقونة على شاشة الهاتف أو سطح المكتب (Installable).
- يعمل دون اتصال بالإنترنت (Offline Capability) لعرض البيانات المحفوظة والـ Resume بفضل تقنية الـ Service Worker.
- سرعة تحميل فورية نتيجة الكاش المحلي للأصول الثابتة والصور والخطوط.

### 4. مدونة ثابتة خفيفة الوزن (Static Blog Section)
قسم المدونة يعمل بالكامل بطريقة ساكنة (Static MDX/JSON)، حيث تتم قراءة المقالات من ملفات محلية دون الحاجة لقاعدة بيانات، مما يجعل تحميل المقالات فورياً ومثالياً لمحركات البحث (SEO).

### 5. نموذج اتصال تفاعلي بدون سيرفر (Contact Form via EmailJS)
يتيح للزوار وأصحاب العمل إرسال رسائل مباشرة إلى البريد الإلكتروني الخاص بك (`kamal.sroor.dev@gmail.com`) من خلال الـ API الخاص بـ EmailJS مع التحقق من صحة البيانات المدخلة وإظهار حالات النجاح والفشل بحركات تفاعلية جذابة.

### 6. تهيئة ممتازة لمحركات البحث (Technical SEO)
بناء الأكواد متوافق تماماً مع متطلبات محركات البحث للظهور في النتائج الأولى:
- هيكلية HTML5 دلالية كاملة.
- وسوم meta وصور Open Graph مخصصة لكل صفحة ومشروع.
- ملفات `sitemap.xml` و `robots.txt` يتم توليدها تلقائياً.
- هيكل بيانات Schema.org لتسهيل فهم محركات البحث لشخصية وخبرات صاحب الموقع.

## 🛠️ البنية البرمجية والتقنيات المستخدمة (Tech Stack)

### الواجهة الأمامية (Frontend)
- **Next.js 15 (App Router):** لاستغلال قوة التطوير الحديث وأفضل ممارسات الأداء، مع تفعيل خاصية التصدير الساكن الكامل (`output: 'export'`).
- **React 19 & TypeScript:** لضمان كتابة كود بروتوكولي آمن وقابل للتوسيع مع تفادي الأخطاء البرمجية أثناء البناء (Build-time errors).
- **Tailwind CSS v4:** لبناء وتخصيص نظام التصميم والألوان بشكل مرن وسريع عبر الـ Utility Classes.
- **Framer Motion:** لتنفيذ الحركات والتنقلات السلسة ومؤثرات الظهور والـ Parallax بشكل جمالي راقٍ.

### الأدوات والخدمات الطرف الثالث (Third-Party Integrations)
- **EmailJS SDK:** لإرسال رسائل الـ Contact Form مباشرة للبريد الإلكتروني دون الحاجة لكود خادم.
- **next-pwa:** لإنشاء الـ Service Worker وإدارة ملف الـ Manifest لتشغيل الـ PWA تلقائياً.
- **Lucide React:** لتوفير مكتبة أيقونات تقنية متناسقة وعالية الجودة ومتوافقة مع إمكانية الوصول.

## 🚀 النشر والرفع على استضافة مشتركة (Shared Hosting / cPanel Deployment)

بما أن الموقع لا يحتوي على خادم خاص (Static Export)، فإن عملية رفعه على الاستضافات المشتركة مثل cPanel بسيطة للغاية ولا تحتاج لأي تطبيقات إضافية:

### الخطوة 1: بناء وتصدير الموقع محلياً
قم بتشغيل الأمر التالي في سطر الأوامر بمجلد المشروع:
```bash
npm run build
```
هذا الأمر سينتج مجلداً جديداً باسم `out/` يحتوي على ملفات HTML و CSS و JavaScript الساكنة بالكامل، بالإضافة لملفات public (بما فيها ملف `.htaccess` و `manifest.json` و الـ Resume).

### الخطوة 2: تهيئة ملف `.htaccess` (مرفق وجاهز)
يحتوي مجلد `public/` على ملف `.htaccess` تم إنشاؤه مسبقاً، ويقوم بالتالي عند رفعه للاستضافة:
1. تحويل الزوار تلقائياً لبروتوكول **HTTPS** الآمن.
2. تفعيل **Gzip Compression** لضغط أصول الموقع وتوفير الباندويث وتسريع التصفح.
3. إدارة الروابط الـ Clean (Trailing Slashes): حيث يقوم خادم Apache بقراءة روابط Next.js كأدلة فرعية (مثل `/about/` بدلاً من `/about.html`) لضمان عدم ظهور خطأ 404 عند عمل تحديث للصفحة.
4. إعداد كاش متصفح الزائر للأصول الثابتة (Expires Headers) لتسريع التحميل في الزيارات التالية.

### الخطوة 3: رفع الملفات إلى cPanel
1. قم بضغط محتويات المجلد `out/` (الذي ينتج بعد عملية البناء) إلى ملف بصيغة `.zip`.
2. ادخل إلى لوحة التحكم cPanel الخاصة باستضافتك.
3. افتح **File Manager** (مدير الملفات) وتوجه إلى المجلد الرئيسي للموقع وهو عادةً **`public_html`**.
4. قم برفع ملف الـ `.zip` الذي قمت بضغطه.
5. قم بفك ضغط الملف (Extract) مباشرة داخل `public_html`.
6. تأكد من ظهور الملفات مباشرة في المسار الرئيسي، بما في ذلك ملف `.htaccess` (ملاحظة: تأكد من تفعيل "Show Hidden Files" في cPanel لرؤية ملفات النقطة مثل `.htaccess`).
