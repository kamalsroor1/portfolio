# خطة العمل وجدول المهام (PLANNING.md)

يوضح هذا الملف خطة التطوير البرمجي المقسمة إلى **9 مراحل متتالية** لإنشاء الموقع بالكامل وتكوينه كـ Static Export متوافق مع PWA للرفع على استضافة مشتركة.

---

## 📅 مراحل التطوير (Development Roadmap)

### 📌 المرحلة 1: إعداد بيئة العمل والتهيئة الأساسية (Setup & Base Configs)
- [x] إنشاء مشروع Next.js 15 مع TypeScript و Tailwind CSS و ESLint.
- [ ] تثبيت الحزم الإضافية اللازمة:
  ```bash
  npm install framer-motion lucide-react clsx tailwind-merge next-pwa
  ```
- [ ] تهيئة ملف `next.config.ts` ليدعم التصدير الساكن الكامل وصور بدون خادم وتفعيل الـ Trailing Slashes لملائمة الاستضافة المشتركة:
  ```typescript
  // next.config.ts
  import { NextConfig } from 'next';
  const nextConfig: NextConfig = {
    output: 'export',
    images: { unoptimized: true },
    trailingSlash: true, // ضروري جداً للاستضافة المشتركة لتوليد مجلدات وسيطة بدلاً من ملفات html مفردة
  };
  export default nextConfig;
  ```
- [x] إضافة ملف `.htaccess` مخصص داخل المجلد `public/` ليقوم خادم Apache بتحويل الروابط تلقائياً ودعم خاصية الـ SSL وتفعيل الكاش للمتصفح.

### 📌 المرحلة 8: تصدير الموقع والرفع (Build & Deployment)
- [ ] تشغيل أمر البناء للحصول على الملفات الساكنة:
  ```bash
  npm run build
  ```
- [ ] التأكد من إنشاء مجلد المخرجات `/out` الذي يحتوي على ملفات HTML الساكنة.
- [ ] **طريقة الرفع على الاستضافة المشتركة (cPanel / Shared Host):**
  1. ضغط محتويات مجلد `/out` بالكامل في ملف ZIP.
  2. الدخول إلى لوحة تحكم الاستضافة cPanel ثم File Manager.
  3. التوجه لمجلد الويب الرئيسي `public_html/`.
  4. رفع ملف الـ ZIP وفك الضغط داخله مباشرة.
